## 触发条件
用户说"生成日报"、"截图汇报"、"分析工作状态"、"工作日报"等。

## 执行步骤

1. **截图识别**：调用 `capture_image` 截取当前屏幕保存到临时文件，再调用 `parse_image_to_markdown` 将截图内容提取为 Markdown 格式文字
2. **视觉分析**（可选）：如果需要理解 UI 布局，调用 `analyze_screen` 通过视觉模型进一步分析
3. **生成报告**：综合识别和视觉分析的结果，生成结构化的 Markdown 日报，包含：
   - 当前时间
   - 屏幕/应用内容摘要
   - 关键信息提取
4. **保存文件**：调用 `write_file` 将报告保存到 `./reports/daily_report_{date}.md`
5. **通知用户**：告知报告生成完成，并显示报告摘要

## 注意事项
- 如果用户指定了特定应用，在截图时使用 `--window` 参数指定窗口
- 报告格式应清晰，使用 Markdown 标题和列表
